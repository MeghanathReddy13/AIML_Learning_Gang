Linear Regression Evaluation

A step-by-step teaching deck using a small sample dataset.

Goal: understand when validation happens, how metrics are calculated, and when regularization is used.

We will predict house price from house size.

1. What is the model trying to learn?

For a simple linear regression model, we assume the relationship looks like this:

Predicted Price = Intercept + Slope × Size

In machine learning notation:

ŷ = β₀ + β₁x
  • x = feature, such as house size
  • ŷ = predicted target value
  • β₀ = intercept
  • β₁ = slope or coefficient

3. Why split the data?

Train data → Learn coefficients

Validation data → Compare model choices

Test data → Final unbiased evaluation

The model should not see validation or test rows while learning coefficients.

Important: validation checks whether the trained model performs well on data it did not learn from.

4. Train the model

Using only the 8 training rows, linear regression learns:

Predicted Price = -1.845 + 0.1485 × Size

Interpretation:

  • The model predicts about $148.50 increase in price per additional square foot.
  • The intercept is a mathematical anchor. It is not always meaningful in business terms.

5. How did the model learn the line?

The model tries many possible lines and chooses the one with the smallest squared error on training data.

Training Objective = minimize Σ(Actual − Predicted)²

For each training row:

Actual price - Predicted price = Error
Squared error = Error²

The best line is the one with the smallest total squared error.

6. After training, coefficients are frozen

Once the model has learned this equation:

ŷ = -1.845 + 0.1485 × Size

Validation starts.

During validation, the model does not update β₀ or β₁. It only predicts and gets judged.

Now we calculate validation metrics.

10. MAE on Validation Data

MAE means Mean Absolute Error.

MAE = average of absolute errors
MAE = (4.077 + 4.970) / 2 = 4.524

Because prices are in thousands:

Validation MAE ≈ $4,524

Meaning: on average, the model is off by about $4,524 on validation data.

11. RMSE on Validation Data

RMSE means Root Mean Squared Error.

RMSE = √ average(Error²)
RMSE = √((4.077² + 4.970²) / 2) = 4.546
Validation RMSE ≈ $4,546

RMSE punishes large errors more than MAE.

12. R² on Validation Data

R² compares the model against a simple baseline.

Baseline = predict the average actual price every time

Validation actual prices:

225 and 315

Average actual price:

(225 + 315) / 2 = 270

If the model cannot beat this baseline, it is not useful.

13. R² Formula

R² = 1 − SSE / SST
  • SSE = model squared error
  • SST = baseline squared error

For this validation set:

Validation R² ≈ 0.990
The model explains about 99% of the variation in validation prices.

20. Why not use test set during model development?

If you repeatedly choose models based on test performance, the test set becomes part of the development process.

Then test performance is no longer unbiased.

Correct usage:

Train → learn
Validation → select/tune
Test → final report

21. Residual analysis

A residual is:

Residual = Actual − Predicted

Good residual behavior:

  • Centered around zero
  • No curve pattern
  • No funnel shape
  • No extreme outliers dominating the model

22. Why residuals matter

Metrics may look good, but residuals can reveal problems.

Residual PatternPossible Issue
Curved patternRelationship may be nonlinear
Funnel shapeError variance changes with prediction size
Large isolated residualsOutliers or data quality issue
Sequential patternAutocorrelation, common in time series

23. Where adjusted R² fits

R² often increases when you add features, even weak features.

Adjusted R² penalizes unnecessary features.

24. Now introduce regularization

Regularization is used when the model is too flexible or coefficients become too large.

Regularized Objective = Prediction Error + Penalty for Large Coefficients

It helps reduce overfitting by keeping the model simpler.

25. Ordinary regression vs regularized regression

Ordinary Linear Regression

Only minimizes prediction error.

minimize SSE

Coefficients can become large and unstable.

Regularized Regression

Minimizes prediction error plus penalty.

minimize SSE + penalty

Coefficients are controlled.

26. Ridge Regression

Ridge uses L2 regularization.

Loss = SSE + λΣβ²

What it does:

  • Shrinks coefficients toward zero
  • Usually does not make them exactly zero
  • Works well when features are correlated

27. Lasso Regression

Lasso uses L1 regularization.

Loss = SSE + λΣ|β|

What it does:

  • Shrinks coefficients
  • Can make some coefficients exactly zero
  • Useful for feature selection

28. Elastic Net

Elastic Net combines Ridge and Lasso.

Loss = SSE + λ₁Σ|β| + λ₂Σβ²

Use it when:

  • You have many features
  • Some features are correlated
  • You want both stability and feature selection

29. How validation chooses regularization strength

λ controls how strongly coefficients are penalized.

Select λ = 1.0 because validation RMSE is lowest.

30. When should students use regularization?

  • When train performance is much better than validation performance
  • When there are many features
  • When features are highly correlated
  • When coefficients are unstable or unusually large
  • When the model performs poorly on new data
For production ML, regularization is often a default starting point rather than an optional advanced technique.

31. Bias-variance tradeoff

No regularization

  • Low bias
  • High variance
  • Can overfit

Too much regularization

  • High bias
  • Low variance
  • Can underfit

Goal: find the middle point where validation error is lowest.

32. End-to-end classroom workflow

1. Prepare dataset
2. Split into train, validation, test
3. Train model on train data
4. Freeze model coefficients
5. Predict on validation data
6. Calculate MAE, RMSE, R², adjusted R²
7. Compare models and regularization settings
8. Select best model
9. Evaluate final model once on test data
10. Review residuals and business reasonableness

33. Student mental model

Training teaches the model.

Validation coaches the model development process.

Testing grades the final model.

Validation does not change coefficients directly. It tells us whether our modeling choices are working.

34. Final Takeaways

  • Linear regression learns coefficients from training data.
  • Validation starts after training and uses unseen data.
  • MAE explains average error in business units.
  • RMSE highlights large mistakes.
  • R² measures improvement over a baseline average prediction.
  • Adjusted R² helps compare models with different numbers of features.
  • Regularization reduces overfitting by penalizing large coefficients.
  • Ridge shrinks; Lasso selects; Elastic Net combines both.

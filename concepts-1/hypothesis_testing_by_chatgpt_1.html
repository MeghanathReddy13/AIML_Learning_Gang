with Sample Datasets

A simple, step-by-step way to teach students why these concepts exist.

Main idea: Is the pattern in our data a real signal, or could it be random noise?

The question students should remember

“Is what I observed bigger than what random chance normally produces?”

Observed pattern

Example: churned customers made more support calls.

Random chance

Maybe this sample is unusual and the pattern is not real.

Why average alone is not enough

Two datasets can have similar average differences but very different uncertainty.

Low variation

Stayed: 1, 2, 1, 3, 2

Churned: 5, 6, 7, 5, 4

Groups are clearly separated.

High variation

Stayed: 1, 10, 0, 8, 2

Churned: 4, 7, 5, 9, 2

Groups overlap a lot.

Hypothesis testing uses both: difference and variation.

Population vs sample

Population

All customers we care about.

Example: 1,000,000 telecom customers.

Sample

The few customers we actually measured.

Example: 10 customers in our small table.

Hypothesis testing helps us use a sample to make a careful statement about the population.

Null and alternative hypotheses

Null hypothesis: H₀

Default assumption.

Support calls are not different between churned and stayed customers.

Alternative hypothesis: H₁

What we are looking for.

Support calls are different between churned and stayed customers.

We start with H₀ and ask: does the data strongly disagree?

Think like a courtroom

Court

Start assumption: innocent.

Need strong evidence to convict.

Statistics

Start assumption: no effect.

Need strong evidence to reject H₀.

Failing to reject H₀ does not prove H₀ is true. It means we do not have enough evidence.

What is standard deviation?

Standard deviation tells us how spread out individual values are.

Small spread

2, 2, 3, 2, 3

Values are close together.

Large spread

0, 9, 2, 10, 1

Values jump around a lot.

More spread means more uncertainty.

Our churn sample: spread

GroupValuesMeanStandard Deviation
Stayed1, 2, 1, 3, 21.80.84
Churned5, 6, 7, 5, 45.41.14

The groups are not only different; each group also has low spread. That makes the difference more believable.

What is standard error?

Standard error tells us how much a sample average may move if we take another sample.

Standard Error = s / √n

s

Spread in the data.

More spread → more uncertainty.

n

Sample size.

More data → less uncertainty.

Why sample size matters

5 customers

Averages can change a lot if we sample again.

Less confidence.

5,000 customers

Averages are more stable.

More confidence.

More data → smaller standard error → stronger evidence

What is a t-test?

A t-test compares the observed difference against the expected random variation.

t = Observed Difference / Expected Random Variation
Signal ÷ Noise = t-value

Our churn t-test result

ValueMeaning
Mean difference = 3.6Churned customers made 3.6 more calls on average.
t-value = 5.69The difference is large compared with expected noise.
p-value = 0.0006This result is very unlikely if H₀ were true.

Decision: Reject H₀. Support calls look meaningfully different between groups.

What does the p-value mean?

p-value answers:

“If there were really no difference, how often would we see a sample result this extreme?”

Small p-value

Result is surprising under H₀.

Evidence against H₀.

Large p-value

Result is not surprising under H₀.

Weak evidence.

Decision rule: alpha

Alpha is the cutoff we choose before testing.

Common alpha: α = 0.05

If p ≤ 0.05

Reject H₀.

Evidence is strong enough.

If p > 0.05

Fail to reject H₀.

Evidence is not strong enough.

Types of t-tests

One-sample

Compare one sample to a known value.

Delivery time vs 30 minutes.

Two-sample

Compare two independent groups.

Churned vs stayed customers.

Paired

Compare before/after for same subject.

Blood pressure before and after medicine.

Do we test every feature?

Sometimes yes, especially during exploratory analysis and simple feature selection.

For each feature, ask:

Age

Are ages different for churned vs stayed?

Income

Are incomes different for churned vs stayed?

Support Calls

Are call counts different for churned vs stayed?

Example feature test results

FeatureTest ideap-valueSimple interpretation
AgeCompare average age0.20Weak evidence
IncomeCompare average income0.04Some evidence
Support CallsCompare average calls0.0006Strong evidence

Support Calls looks like a stronger signal than Age in this sample.

Important: correlation is not causation

Hypothesis testing can show that churned customers had more support calls.

It does not prove support calls caused churn.

Maybe customers called because they already had service problems. The real cause may be product quality, billing issues, or network outages.

Regression also uses t-tests

In linear regression, each feature gets a coefficient.

Y = β₀ + β₁Age + β₂Income + β₃Calls

For every coefficient, regression asks:

H₀

β = 0

Feature adds no value.

H₁

β ≠ 0

Feature contributes something.

Regression p-values answer: does this feature still matter after considering other features?

Simple words: the result is promising, but not strong enough under the 5% rule. We may need more data.

Errors in hypothesis testing

Type I Error

False positive.

We say there is an effect, but there is no real effect.

Type II Error

False negative.

We miss a real effect.

Good testing balances both risks.

When do ML learners need this?

Before modeling

Understand data and identify possible useful features.

During modeling

Interpret coefficients and decide which predictors matter.

After modeling

Check whether model improvement is meaningful.

In production

Monitor drift, A/B tests, and business changes.

Final mental model

Hypothesis testing is a careful way to ask:

Signal > Noise?

Difference

What changed?

Uncertainty

How noisy is the data?

Decision

Is evidence strong enough?

Machine learning finds patterns. Hypothesis testing helps us decide whether patterns are likely real or just random.

Linear Regression vs Deep Neural Networks

A student-friendly explanation using one dataset: how models learn from features, how hidden layers create learned representations, and how linear learning becomes nonlinear.

1. The Learning Problem

We want a model to predict house price from known input features.

Size sq ftBedroomsAge yearsPrice $1000s

Input features: Size, Bedrooms, Age. Target: Price.

2. What Linear Regression Learns

Linear regression learns one weighted equation:

Price = w₁(Size) + w₂(Bedrooms) + w₃(Age) + b

Example learned model:

Price = 0.18(Size) + 40(Bedrooms) - 2(Age) - 80

Each feature directly contributes to the final prediction.

3. Linear Regression Prediction Example

For a house with Size = 1600, Bedrooms = 3, Age = 8:

Price = 0.18(1600) + 40(3) - 2(8) - 80
Price = 288 + 120 - 16 - 80 = 312

Predicted price = $312,000

4. What Do Linear Weights Mean?

Size weight

0.18

Larger houses increase predicted price.

Bedroom weight

40

More bedrooms increase predicted price.

Age weight

-2

Older houses decrease predicted price.

Linear regression is easy to interpret because the model directly uses the original features.

5. Limitation of Linear Regression

Linear regression works well when the relationship is close to a straight-line pattern.

Price ^ | * | * | * | * | * +--------------------> Size

If the real pattern bends, jumps, or depends on feature combinations, one linear equation may not fit well.

6. Neural Network: Same Inputs, Different Learning

A neural network also starts with the same input features.

Size --------\ Bedrooms ----- > Hidden Layer(s) ----> Price Age ----------/

Important: The neural network cannot use information that is not in the input data.

7. Hidden Layers Do Not Create New Data

If the dataset contains only:

X = [Size, Bedrooms, Age]

then every hidden layer is built from those features.

A hidden layer creates new representations of the existing information, not new external information.

8. How One Hidden Neuron Works

A hidden neuron computes a weighted combination:

z = w₁(Size) + w₂(Bedrooms) + w₃(Age) + b

Then it applies an activation function:

h = activation(z)

The output h becomes a new learned feature for the next layer.

9. Example: A Learned Feature

Suppose a hidden neuron learns:

h₁ = ReLU(0.8(Size) + 1.5(Bedrooms) - 2(Age))

This might behave like a:

LuxuryScore = Large size + many bedrooms - old age

The network does not name it LuxuryScore. We give it that name to understand its behavior.

10. Multiple Hidden Neurons Learn Multiple Patterns

Neuron 1

h₁ = ReLU(0.8Size + 1.5Bedrooms - 2Age)

Luxury-like pattern

Neuron 2

h₂ = ReLU(-0.3Size + 2Bedrooms)

Bedroom-heavy pattern

Neuron 3

h₃ = ReLU(-Age)

Newness pattern

Neuron 4

h₄ = ReLU(Size - Age)

Premium-like pattern

11. Why We Call Them “New Features”

Original features:

Size, Bedrooms, Age

Learned features:

LuxuryScore, ModernityScore, FamilyHomeScore

They are not new columns from the outside world. They are useful transformations of the original columns.

Hidden layers are like combining ingredients. They do not create new ingredients; they create useful combinations.

13. How Does Learning Improve These Features?

At the beginning, weights are random, so hidden features are mostly useless.

Random weights ↓ Bad predictions ↓ Prediction error ↓ Backpropagation adjusts weights ↓ Useful hidden features

14. Backpropagation in Simple Words

Backpropagation asks:

1

How wrong was the prediction?

2

Which weights contributed to the error?

3

How should weights change to reduce future error?

Over many updates, neurons become detectors for useful patterns.

15. Linear + Linear Is Still Linear

Suppose we stack two linear layers without activation:

h = W₁x + b₁
y = W₂h + b₂

Substitute h into y:

y = W₂(W₁x + b₁) + b₂ = Wx + b

Many linear layers collapse into one linear layer.

16. So How Does Linear Become Nonlinear?

The activation function makes the network nonlinear.

Linear step: z = Wx + b
Nonlinear step: h = ReLU(z)

Linear + Activation + Linear + Activation = Nonlinear model

17. ReLU Activation

ReLU means:

ReLU(z) = 0 if z < 0, otherwise z
Output ^ | / | / | / |_____/ | +----------------> z

ReLU creates a bend. That bend is what breaks pure linearity.

18. ReLU as a Threshold Detector

19. Nonlinear House Price Pattern

Suppose luxury pricing jumps when a house is large and has many bedrooms.

SizeBedroomsPrice
18003300
19003320
21004600
22004650

The important pattern is not Size alone or Bedrooms alone. It is their combination.

20. Linear Regression Struggles Here

Linear regression tries to explain the data with steady change.

Price ^ | * luxury | | * | * normal | * +----------------------> Size

A single straight line cannot easily model a sudden luxury premium.

21. Neural Network Can Learn the Jump

Hidden neurons can learn detectors like:

h₁ = ReLU(Size - 2000)
h₂ = ReLU(Bedrooms - 3)

Then the output layer combines them:

Price = base + 2h₁ + 3h₂ + luxury adjustment

This allows the model to represent nonlinear rules.

22. Deep Layers Build Feature Hierarchies

Input Features Size, Bedrooms, Age ↓ Layer 1 Large house, New house, Many bedrooms ↓ Layer 2 Luxury home, Family home, Modern home ↓ Output Predicted price

23. Image Example: Same Idea

In image recognition, the raw input is pixels.

Pixels ↓ Edges, lines, corners ↓ Eyes, nose, mouth ↓ Face ↓ Prediction

Each layer transforms simple inputs into more meaningful learned features.

25. The Most Important Takeaways

Linear Regression

Finds one best weighted equation from input features to prediction.

Hidden Layer

Creates learned combinations of existing input features.

Activation

Introduces nonlinearity. Without it, deep linear layers collapse into one linear model.

Deep Learning

Builds layers of representations that make complex prediction easier.

26. Final Mental Model

Linear Regression

Features ↓ Weighted sum ↓ Prediction

Deep Neural Network

Features ↓ Learned features ↓ Higher-level features ↓ Prediction

Deep learning does not create new information. It learns better representations of the information already present in the input.

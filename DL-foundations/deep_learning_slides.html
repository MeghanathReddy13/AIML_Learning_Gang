02 · Why Linear Regression Isn't Enough
The Linear Model Hits a Ceiling
Even with all 79 features, linear regression can only draw straight lines through a curved world.
LINEAR REGRESSION MODEL
SalePrice = β₀ + β₁·LotArea + β₂·Quality + β₃·GrLivArea + ...
▸ Assumes features combine additively
▸ Cannot model interactions between features
▸ One straight line per feature
⚠️
Real Test Results
Linear Regression on all 79 features:
RMSE: $34,000 — off by $34K on average
R² = 0.79 — explains only 79% of variance
It still misses luxury homes by $80–120K
THE PATTERN WE CAN'T CAPTURE
A 3,000 sqft house in a good neighborhood with a 9-quality rating is not just the sum of its parts. The interaction between those three factors creates exponential value — a non-linear relationship.

03 · The Solution
Neural Networks: Universal Approximators
A neural network learns which combinations of features matter — and by how much — directly from data.
■ Input features   ■ Hidden layers (learn interactions)   ■ Output (price)
HOW IT WORKS
🔵 Input Layer — raw features (LotArea, Quality, GrLivArea…)
🟣 Hidden Layers — learn combinations: "high quality + big house + good area" → high value
🟠 Output Layer — predicted SalePrice
⚡ Each connection has a weight the model learns during training
THE KEY INSIGHT
The hidden layer neurons fire in combinations. One neuron might learn "luxury indicator" — activating when quality is high AND neighborhood is elite AND size is large.

This is the non-linearity linear regression could never capture.
UNIVERSAL APPROXIMATION THEOREM
A neural network with one hidden layer can approximate any continuous function to arbitrary accuracy — given enough neurons.
04 · Non-Linearity
Activation Functions: The Secret Ingredient
Without an activation function, stacking layers is pointless — it's still just linear regression.
🔴
Without Activation
Layer2 = W2 · (W1 · x + b1) + b2
= (W2·W1)·x + (W2·b1 + b2)
= W_combined · x + b_combined
Still just linear! 10 layers = 1 layer.
✅
With ReLU (for our house prices)
ReLU(x) = max(0, x)
Each neuron "fires" only when relevant — e.g., the quality interaction only activates above a threshold.
This is what creates non-linear decision boundaries.
→ CLICK BUTTONS TO COMPARE
For house price regression: ReLU works best in hidden layers (fast, no vanishing gradient). Output layer: no activation (we want any real number).
05 · Learning
How the Network Learns: Gradient Descent
The network adjusts its weights iteratively — moving downhill on the loss surface toward minimum error.
Learning Rate: α = 0.03
THE UPDATE RULE
w ← w − α · ∂Loss/∂w
weight = weight − learning_rate × gradient
🔴
α too large (0.1+)
Overshoots — oscillates or diverges. Loss explodes to NaN.
✅
α = 0.001–0.01
Smooth convergence. Loss decreases steadily each epoch.
🐢
α too small (0.00001)
Converges, but takes thousands of epochs. Painfully slow.
06 · Backpropagation
Backprop: Blame Assignment
"How much did this particular weight contribute to our error?" — Chain rule answers this for every weight simultaneously.
THE CHAIN RULE IN ONE HOUSE
Predicted: $320,000  ·  Actual: $250,000
Error = +$70,000

Backprop asks: Which hidden neuron pushed the output too high?
Which input weight fed that neuron too strongly?
Adjust all of them proportionally.
# Simplified backprop for house price loss = (predicted - actual) ** 2 # MSE dL_dOut = 2 * (predicted - actual) # ∂Loss/∂output dL_dW2 = dL_dOut * hidden_activations # ∂L/∂W2 dL_dH = dL_dOut * W2 # propagate back dL_dW1 = dL_dH * relu_grad * inputs # ∂L/∂W1
WHY IT'S EFFICIENT
Backprop computes ALL gradients in a single backward pass — the same cost as one forward pass. For a network with millions of weights, this is what makes training feasible.

08 · The Next Trap
Overfitting: Memorizing, Not Learning
A deep network can memorize all 1,460 houses — and fail completely on houses it's never seen.
😱
Classic Overfitting Signature
Train loss → near zero
Validation loss → keeps rising

The model memorized noise, not the signal.
THE FIXES
🛑 Dropout — randomly zero out neurons during training
📏 L2 Regularization — penalize large weights
🔁 Early Stopping — stop when val loss starts rising
📊 More data / data augmentation
DROPOUT IN CODE
model = Sequential([ Dense(128, activation='relu'), Dropout(0.3), # kill 30% of neurons Dense(64, activation='relu'), Dropout(0.2), Dense(1) ])
